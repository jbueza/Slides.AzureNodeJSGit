A few things...

  • Custom Domains: coming soon to Shared. Works in Reserved though!
  • SSL: works great on azuresites.net, not so great on custom domains in reserved mode.
  • HTTP Requests through IIS: no web sockets out of the box, must use worker role

The Good Parts

  • Convenience: if you wanted a site up and running very quickly, this is how you do it!
  • Cheap: shared websites (you get 10) are free! Reserved instances cost money.
  • Scale: vertically or horizontally!
  • Potential: Azure is evolving rapidly, more services, tools, and APIs coming out every few weeks
  • Tooling: easily slot in Jenkins to run your unit tests, service layer tests, and then push your site
  • Open: deploy any stack you want (NodeJS, PHP, JavaScript, Python)
